## Design Context

### Users
Indie game developers using the tool in a production workflow to clean up generated or uneven sprite sheets and prepare game-ready assets.

### Brand Personality
Retro game-tool. Utility-focused, production-oriented, and clearly not generic AI slop.

### Aesthetic Direction
Dark workspace UI with retro game-tool energy. The interface should feel purpose-built for frame cleanup and export rather than like a generic dashboard or AI-generated admin template.

### Design Principles
1. Prioritize precise sprite cleanup over decorative UI.
2. Make frame editing and export intent obvious at a glance.
3. Preserve a retro-tool character without sacrificing usability.
4. Avoid generic AI UI tells and interchangeable dashboard patterns.
5. Support game-asset workflows with clear, production-ready controls.
